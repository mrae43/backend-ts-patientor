# Patientor Backend

A TypeScript-based Express backend for the Patientor medical record application. This application provides API endpoints for managing patient diagnoses and health information.

## Overview

Patientor is a simple medical record application designed for doctors to:

- Manage patient diagnoses
- Track basic health information
- Store and retrieve patient medical records
- Handle medical data securely and efficiently

This backend service provides the core REST API for these operations.

## Project Structure

```
patientor-backend-ts/
├── src/
```

## Tech Stack

- **Language:** TypeScript 5.9.3
- **Runtime:** Node.js 25.x
- **Web Framework:** Express 5.2.1
- **Package Manager:** pnpm 10.28.0
- **Build Tool:** TypeScript Compiler (tsc)
- **Linting:** ESLint with TypeScript support

## Prerequisites

- Node.js 18.x or higher
- pnpm 10.x or higher (or npm/yarn as alternatives)

## Installation

1. Clone the repository:

```bash
git clone <repository-url>
cd patientor-backend-ts
```

2. Install dependencies:

```bash
pnpm install
```

## Available Scripts

### Development

Start the development server with hot-reload:

```bash
pnpm dev
```

The server will run on `http://localhost:3000` by default.

### Building

Compile TypeScript to JavaScript:

```bash
pnpm build
```

Compiled files will be output to the `build/` directory.

### Production

Run the compiled application:

```bash
pnpm start
```

### Linting

Check code quality with ESLint:

```bash
pnpm lint
```

### Type Checking

Validate TypeScript without emitting:

```bash
pnpm exec tsc --noEmit
```

## API Endpoints

**Response:**

```
pong
```

## Configuration

### TypeScript Configuration (`tsconfig.json`)

- **Target:** ESNext
- **Module:** nodenext (ES modules)
- **Strict Mode:** Enabled
- **Source Maps:** Enabled for debugging
- **Unused Variables:** Errors on unused locals/parameters

### Environment

The application runs with the following defaults:

- **Port:** 3000
- **Module Type:** ES Modules (configured in `package.json`)

## Development Workflow

1. **Write/modify code** in the `src/` directory
2. **Type check** with TypeScript: `pnpm exec tsc --noEmit`
3. **Lint code** with ESLint: `pnpm lint`
4. **Test in development** with: `pnpm dev`
5. **Build for production** with: `pnpm build`
6. **Run production build** with: `pnpm start`
